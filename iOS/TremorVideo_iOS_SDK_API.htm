API Code

TremorVideoAd

TremorVideoAd exists as a static class containing the methods you use to interact with the SDK.

 

+ (void)initWithAppID:(NSString *)appID

This method is used to initialize the TremorVideo static instance. It requires that you pass your custom appID.  It is recommended that this be called at your application launch.

 

Parameters

appID

       Your application’s unique appID.

 

+ (void)loadAd

Use the 'loadAd’ method to fetch a video ad. This method communicates over the network to request ads and download them in the background. You must call this method if you wish to show any ads. Callback “adReady” will be called when loadAd completes.

 

+ (BOOL)showAd:(UIViewController *)parentViewController

‘showAd’ attempts to show an ad on your behalf.

 

Parameters

parentViewController

            The video ad will display in a subview of this UIViewController. This UIViewController should be the UIViewController at the top of hierarchy of current Window.

 

            Return Value

true if the Tremor SDK displayed to screen. Otherwise false.  An unsuccessful attempt to display the ad may be the result of a no ad match, a download in progress, or an error.

 

+ (void)stop

‘stop’ is used to suspend the ad request and download process. This method works asynchronously to suspend the background thread. The results of calling stop may not take effect immediately. Use this method to stop the TremorVideo SDK from using any CPU and network resources.

 

+ (void)destroy

‘destroy’ is used to shutdown any background processes and to perform cleanup of the SDK.  It is recommended that this be called when you are terminating your application.

 

+ (void)setDelegate:(id<TremorVideoDelegate>)delegate

Set a delegate to track changes of Ad state. ‘initWithAppID’ must be called before calling ‘setDelegate’.

 

Parameters

delegate

       Your delegate object.

 

 

+ (BOOL) isAdReady

‘isAdReady’ is used to check if a video ad is ready to be shown. (If you are showing a Ad in a frame with  -showAd : withFrame, use ‘isAdreadyFrame’ instead.

 

Return Value

true if a video ad is ready to be shown. Otherwise false.

 

+ (TremorVideoSettings *)getSettings

‘getSettings’ returns a TremorVideoSettings object.  You can set properties of this object to update targeting information and other custom settings you’d like to use.

 

Return Value

A pointer to the TremorVideoSettings object.

 

+ (NSString *)getVersion

Return Value

String value of the current Tremor SDK version

 

+ (BOOL)showAd:(UIViewController *)parentViewController withFrame:(CGRect)frame

Attempts to show an ad within the bounds of a CGRect.  This is useful if the ad should not be displayed in full screen.

 

Parameters

parentViewController

            The video ad will display in a subview of this UIViewController.

 

frame

The frame that the video ad should display within.

 

Return Value

true if the ad was successfully displayed, otherwise false.  An unsuccessful attempt to display the ad may be the result of a no ad match, a download in progress, or an error.

 

+ (UIView *)adView

Returns a pointer to the UIView which the ad is displayed in. To show an Ad in a frame using showAd:withFrame, use this method to retrieve a UIView and add it as a subview to your application’s view

 

Return Value

A pointer to the UIView which the ad is displayed in.

+ (void)setFrame:(CGRect)frame

 

Changes the bounds that the video is displayed in. This can be called while the ad is playing and is useful if the bounds of the video must change during screen rotations.  This method can only be used when showing an ad with showAd:withFrame.

 

Parameters

frame

The frame that the video ad should display within.

 

+(void)stopAd

‘stopAd’ is used to stop the ad in frame. Use this method to stop a video ad shown within a frame.

 

+ (BOOL) isAdReadyForFrame

            “isAdReady” checks if a video ad is ready to be shown in a frame.

 

Return Value

true if a video ad is ready to be shown in a frame. Otherwise false.

 

 

 

Callbacks to track changes of Ad state

The Tremor SDK provides the following interface callbacks for you to track the state of an Ad after a request has been made to show one.

 

@protocol TremorVideoAdDelegate <NSObject>

@optional

/*

 * If Ads are being pre-fetched

 * this method will be called when pre-fetching ad is complete

 *

 * Parameters

 * success -

 *      true: ad pre-fetched successfully

 *      false: an error occured during the pre-fetching

 */

- (void)adReady:(BOOL)success;

 

/*

 * This method will be called when an ad is fully loaded

 * and about to start rendering on screen

 */

- (void)adStart;

 

/*

 * This method will be called when an the Tremor SDK has finished with

 * the request to show an Ad.

 *

 * Parameters

 * sucess -

 *      true: ad showed successfully

 *      false: ad was not shown or an error occurred

 *

 * responseCode -

 *      kTremorResponseCodeSuccess:

 *          ad showed succesfully

 *      kTremorResponseCodeNoAd:

 *          No ad was available at the time

 *      kTremorResponseCodeError:

 *          An ad was returned but an error occured tyring to show it

 */

- (void)adComplete:(BOOL)success responseCode:(NSInteger)responseCode;

 

/*

 * This method will be called when the user skips the ad

 */

- (void)adSkipped;

 

/*

 * This method will be called when the user clicks on the ad and open a webpage

 */

- (void)adClickThru;

 

/*

 * This method will be called when an ad video impression event is recorded

 */

- (void)adImpression;

 

/*

 * This method will be called when an ad video starts

 *

 * Parameters

 * videoId -

 *      Name of video ID.

 *      An ad might have multiple videos. The name of first video is "video-1".

 */

- (void)adVideoStart:(NSString *) videoId;

 

/*

 * This method will be called when an ad video hits the first quartile

 *

 * Parameters

 * videoId -

 *      Name of video ID.

 *      An ad might have multiple videos. The name of first video is "video-1".

 */

- (void)adVideoFirstQuartile:(NSString *) videoId;

 

/*

 * This method will be called when an ad video hits midpoint

 *

 * Parameters

 * videoId -

 *      Name of video ID.

 *      An ad might have multiple videos. The name of first video is "video-1".

 */

- (void)adVideoMidPoint:(NSString *) videoId;

 

/*

 * This method will be called when an ad video hits the third quartile

 *

 * Parameters

 * videoId -

 *      Name of video ID.

 *      An ad might have multiple videos. The name of first video is "video-1".

 */

- (void)adVideoThirdQuartile:(NSString *) videoId;

 

/*

 * This method will be called when an ad video ends

 *

 * Parameters

 * videoId -

 *      Name of video ID.

 *      An ad might have multiple videos. The name of first video is "video-1".

 */

- (void)adVideoComplete:(NSString *) videoId;

@end

App Analytics

The SDK has built-in features allowing you to report analytics data for your application. Application session information is automatically calculated when the application enters and resumes from the background. Additional custom data and state information may also be sent.

 

+ (void)handleAnalyticsStateChange:(NSString *)newState;

 

This method is used to register a custom state change in your application.

 

Parameters

newState

            The new state you are going to.

 

+ (void)handleAnalyticsEvent:(NSString *)event

 

This method is used to register a custom event that has occurred in your application.

 

Parameters

event

            The event to register.

 

+ (void)handleAnalyticsEvent:(NSString *)event parameters:(NSDictionary *)parameters

 

This method is used to register a parameterized custom event that has occurred in your application.

 

Parameters

event

            The event to register.

parameters

            A collection of key/value pairs to associate with the event.

 

VAST Display

The SDK has built-in functionality to display VAST ads without going through the Tremor Network.

This is done by calling the following API with VAST URL. Different APIs allow you to customize if skipping the Ad is allowed and if a countdown timer showing the remaining ad time should be displayed. Note it is not necessary to call initWithAppID or loadAd before showing a VAST Ad.

 

+ (BOOL)showVASTAd:(UIViewController *)parentViewController vastURL:(NSString *)url;

 

This will attempt to show the VAST Ad if the supplied url returns a valid VAST xml.

The Ad cannot be skipped.

A count down timer will be shown displaying ad time remaining.

 

Parameters

parentViewController

            The ViewController you are showing the ad from. This UIViewController should be the UIViewController at the top of hierarchy of current Window.

 

url

            The VAST tag url.

 

Return Value

true if the url given is a valid URL and proper parentViewController is passed, otherwise false.

 

 

+ (BOOL)showVASTAd:(UIViewController *)parentViewController vastURL:(NSString *)url skipDelay:(int)skipDelay;

 

This will attempt to show the VAST Ad if the supplied url returns a valid VAST xml.

The Ad can be skipped after a specified delay.

A count down timer will be shown displaying ad time remaining.

 

Parameters

parentViewController

            The ViewController you are showing the ad from. This UIViewController should be the UIViewController at the top of hierarchy of current Window.

 

url

            The VAST tag url.

skipDelay

            Time in seconds after which a skip button will be shown. If 0 no skip button will be shown.

 

Return Value

true if the url given is a valid URL and proper parentViewController is passed, otherwise false.

Return Value

true if the url mentioned is a valid URL and proper parentViewController is passed, otherwise false.

This will attempt to show the VAST Ad if the supplied url returns a valid VAST xml.

The Ad can be skipped after a specified delay.

A count down timer can be optionally shown displaying ad time remaining.

 

Parameters

url

Conversion Tracking

The SDK has a conversion tracking API to allow you to measure the effectiveness of app install advertisements. The conversion tracking event is only fired once per install so that the application doesn’t make additional network calls on subsequent application launches. Note it is not necessary to call initWithAppID or start to use this API.

 

+ (void)fireConversion:(NSString *)advertizer_id :(NSString *)conversion_id;

 

‘fireConversion’ fires the conversion tracking URL to Tremor.

 

Parameters

advertiser_id

            The advertiser id provided by Tremor.

Conversion_id

            The conversion id provided by Tremor.
