API Code

TremorVideo

TremorVideo exists as a static class containing the methods you use to interact with the SDK.

 

public static void initialize( Context context, String appID )

 

This method is used to initialize the TremorVideo static instance.  It is recommended that this method be called when your application launches. Only needs to be called once.

 

Parameters

context

            The context for your application.

appID

            Your applications unique appID.

 

public static void loadAd( ) throws Exception

 

loadAd is used to make a request to fetch a video ad. This method starts a background thread that communicates over the network to request ads and download them in background. You must call this method if you wish to show any ads. Callback adReady will be called when loadAd completes.

 

 

public static boolean showAd( Activity activity, int requestCode )

throws Exception

 

showAd attempts to show an ad on your behalf.

When showAd is called there will always be a corresponding callback to adComplete indicating whether an Ad was shown or not.

 

Parameters

activity

            The activity you are showing the ad from.

requestCode

            A value that will be returned to you via your Activitys onActivityResult. Please note that the request code must be non-negative as per the Android SDK documentation.

 

Return Value

true if the Tremor SDK displayed to screen. Otherwise false.  An unsuccessful attempt to display the ad may be the result of a no ad match, a download in progress, or an error.

 

public static void stop( )

 

stop is used to suspend the ad request and download process. This method works asynchronously to suspend the background thread. The results of calling stop may not take effect immediately. Use this method to stop the TremorVideo SDK from using any CPU and network resources. We recommend that this method be called when your application loses focus or when the application session has ended.

 

public static void destroy( )

 

destroy should be called when your Activity is destroyed, preferably in your Activitys onDestroy handler. This is used to shutdown any background processes and to perform some cleanup of the SDK. Failure to call this method may result in the SDK continuing to run in the background on the device.

 

public static boolean isAdReady()

 

isAdReady is used to check if a video ad is ready to be shown. (If you show a video ad in a frame, use isManualAdReady.

 

Return Value

true if a video ad is ready to be shown. Otherwise false.

 

 

public static void updateSettings( Settings settings )

 

updateSettings allows you to update targeting information and other custom settings youd like to use.

 

Parameters

settings

            The settings to apply.

 

public static String getSDKVersion( )

Return Value

String value of the current Tremor SDK version

 

public static AdViewManager showAdManual( Activity activity, ViewGroup container )

throws Exception

 

showAdManual attempts to show an ad inside of the container passed in.

 

Parameters

activity

            The activity you are showing the ad from.

container

            If an ad is shown, it will be inserted into the container passed in. The ad will stretch to fill the bounds by specifying FILL_PARENT for its width and height layout.

 

Return Value

An instance of the AdViewManager class is returned. This class is used to pass event information into the ad so that it may pause and resume itself when system events occur.

 

AdViewManager

AdViewManager is only used when manually showing ads. It has various methods you must call to alert the ad of certain system level events.

 

public void onStop( )

 

This method should be called when the calling Activitys onStop is called.

 

public void onResume( )

 

This method should be called when the calling Activitys onResume is called.

 

public void onScreenOff( )

 

This method should be called when the devices screen is turned off.

 

public void onScreenOn( )

 

This method should be called when the devices screen is turned off and then back on.

 

public void onActivityResult( int requestCode, int resultCode, Intent data )

 

This method should be called when the current Activitys onActivityResult is called.

VAST Display

The SDK has built-in functionality to display VAST ads without going through the Tremor Network. This is done by calling the following api with a link to a valid VAST tag. The different apis allow you to customize if skipping the Ad is allowed and if a countdown timer showing the remaining ad time should be displayed. Note it is not necessary to call initialize or start before showing a VAST Ad.

 

public static boolean showVASTAd( Activity activity, String vastURL,

int requestCode ) throws Exception

 

 showVASTAd  lets you to play a VAST Tag specified in vastURL.You can check Ad Pass/Failure via onActivityResult.

 

Ad wont skip and count-down timer present in Ad.

 

Parameters

activity

            The activity you are showing the ad from.

vastURL

            The vast tag url.

requestCode

            A value that will be returned to you via your Activitys onActivityResult. Please note that the request code must be non-negative as per the Android SDK documentation.

 

Return Value

true if the vastURL mentioned is a valid URL and proper activity is passed, otherwise false.

int requestCode ,int skipDelaySeconds ) throws Exception

 

 showVASTAd  lets you to play a VAST Tag specified in vastURL.You can check Ad Pass/Failure via onActivityResult.

 

Ad will show skip button based on skipDelaySeconds value and count-down timer present in Ad.

 

Parameters

activity

            The activity you are showing the ad from.

vastURL

            The vast tag url.

requestCode

A value that will be returned to you via your Activitys onActivityResult. Please note that the request code must be non-negative as per the Android SDK documentation.

 

skipDelaySeconds

Time in seconds after which a skip button will be shown. If 0 no skip button will be shown.

 

Return Value

true if the vastURL mentioned is a valid URL and proper activity is passed, otherwise false.

Conversion Tracking

The SDK has a conversion tracking API to allow you to measure the effectiveness of app install advertisements. The conversion tracking event is only fired once per install so that the application doesnt make additional network calls on subsequent application launches. Note it is not necessary to call initialize or start to use this API.

 

public static boolean fireConversion( Activity activity, final String advertiser_id, final conversion_id) throws Exception

 

fireConversion fires the conversion tracking URL to Tremor.

 

Parameters

activity

            The activity you are showing the ad from.

advertiser_id

            The advertiser id provided by Tremor.

Conversion_id

            The conversion id provided by Tremor.

 

Return Value

true if firing the conversion tracking URL succeeds; otherwise false.

 

 

 

 

App Analytics

The SDK has built-in features allowing you to report analytics data for your application. Session information is automatically calculated when you start and stop the static TremorVideo background process. Custom data and state information may also be sent manually.

 

public void handleAnalyticsStateChange( Activity activity, String newState )

 

This method is used to register a custom state change in your application.

 

Parameters

activity

            The activity currently executing.

newState

            The new state you are going to.

 

public void handleAnalyticsEvent( Activity activity, String event )

 

This method is used to register a custom event that has occurred in your application.

 

Parameters

activity

            The activity currently executing.

event

            The event to register.

 

public void handleAnalyticsEvent( Activity activity, String event,

Map<String, String> parameters )

 

This method is used to register a parameterized custom event that has occurred in your application.

 

Parameters

activity

            The activity currently executing.

event

            The event to register.

parameters

            A collection of key/value pairs to associate with the event.
